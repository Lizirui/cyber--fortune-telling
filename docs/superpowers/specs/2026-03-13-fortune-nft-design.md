# Fortune NFT - AI 祝福语盲盒 NFT 设计文档

## 1. 项目概述

**项目名称**: Fortune NFT (算命 NFT)

**项目定位**: 一个趣味盲盒 NFT 项目，用户 mint 后可获得由 AI 实时生成的 60-80 字祝福语，稀有度在 mint 后揭晓，为用户带来"开盲盒"的惊喜体验。

**目标用户**: Web3 爱好者、NFT 收藏者、追求有趣数字纪念品的用户

## 2. 功能规格

### 2.1 Mint 盲盒

- 用户连接钱包后，支付固定 mint 费用获得一个 NFT
- NFT 内容为 AI 实时生成的 60-80 字中文祝福语
- 祝福语稀有度在 mint 后揭晓（普通/稀有/史诗/传说/神话）
- 每个 NFT 的祝福语都是独一无二的

### 2.2 稀有度系统

| 稀有度 | 概率 | 描述 |
|--------|------|------|
| 普通 | 50% | 基础祝福语 |
| 稀有 | 25% | 较好的祝福语 |
| 史诗 | 15% | 精美的祝福语 |
| 传说 | 8% | 极好的祝福语 |
| 神话 | 2% | 绝佳的祝福语 |

稀有度在 mint 时由后端服务随机判定，NFT 上链时一并写入合约。

### 2.3 交易市场

- 用户可以将拥有的 NFT 挂单出售，设定价格
- 其他用户可以购买挂单中的 NFT
- 支持在项目网站内完成所有交易流程

### 2.4 OpenSea 兼容

- NFT 遵循 ERC-721 标准
- 可直接在 OpenSea 等第三方市场交易

## 3. 技术架构

### 3.1 区块链层

- **主链**: Base (EVM L2)
- **标准**: ERC-721
- **链上存储**: 祝福语文本 + 稀有度等级

### 3.2 智能合约

**NFT 合约功能**:
- `mint()` - 用户铸造 NFT，支付固定费用
- `listItem()` - 挂单出售
- `buyItem()` - 购买NFT
- `cancelListing()` - 取消挂单
- `tokenURI()` - 返回 NFT 元数据（含祝福语和稀有度）

### 3.3 后端服务

**AI 生成服务**:
- 接收 mint 请求
- 调用 AI API (Claude/GPT) 生成 60-80 字祝福语
- 随机判定稀有度
- 返回祝福语 + 稀有度给前端

**祝福语 Prompt 模板**:
```
请生成一个60-80字的中文祝福语，要求：
1. 积极向上，正能量
2. 可以包含一些玄学/命运相关的元素
3. 语言优美，有诗意
4. 不要使用特殊符号和表情
```

### 3.4 前端 DApp

**页面结构**:
- 首页 - Mint 盲盒入口
- Mint 页面 - 点击 mint、揭晓动画、查看结果
- 市场页面 - 浏览/搜索在售 NFT、购买
- 个人中心 - 查看拥有的 NFT、管理挂单

**核心交互**:
- 钱包连接 (RainbowKit/Wagmi)
- Mint 流程：连接钱包 → 点击 Mint → 等待 AI 生成 → 揭晓动画 → 显示祝福语和稀有度
- 交易流程：选择 NFT → 设定价格挂单 / 购买 → 确认交易

## 4. 数据流

```
用户点击 Mint
    ↓
前端调用后端 AI 服务
    ↓
AI 服务生成祝福语 + 稀有度
    ↓
前端调用合约 mint()，传入祝福语和稀有度
    ↓
交易确认，NFT 铸造成功
    ↓
前端展示揭晓动画
    ↓
显示祝福语和稀有度
```

## 5. 关键设计决策

1. **链上存储祝福语**: 保证 NFT 数据的去中心化和持久性，Base Gas 足够低
2. **后端判定稀有度**: 避免链上随机数被预测，增加公平性
3. **mint 后揭晓**: 增强惊喜感和趣味性
4. **自建市场 + OpenSea 兼容**: 兼顾流量和灵活性
